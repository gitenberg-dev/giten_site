Frequently Asked Questions

Why are we putting books on Github?

In the world of modern software development, Github is the equivalent of the Library of Alexandria. It is a massive collection of software source code consisting of text. This source code is available as Free and Open Source Software (FOSS) with open licensing allowing developers and users to do what they wish with the source code.

On a computer, books and source code look much the same: files with lines of text. We believe that the tools that have enabled collaboration on FOSS can can also be applied to the world of books.

Repositories (which in our case, is a collection of text files with a history that represent a book and its metadata) can be forked. Forking means making a copy of a repository where you are free to make any changes you wish.

More pragmatically, GitHub is free for public open source projects.

Who is this for?

The early versions of GITenberg are going to be mainly for developers, and for people who know how to use git and GitHub. But the books that we curate and publish are for the benefit of all.

What can I do with these books?

Anything you want, really! These books are in the public domain. You may quote them, edit them, print them, sell them, or give them away in whole or in part.

See our license page for details.

There are other nifty other projects that use public domain books. If you know of any (you may, as you're reading this), please let us know what they are so that we can link them here!

Can I sell these books?

Yes, that is an acceptable use of works that are in the Public Domain.

Note that not all of our covers are available for commercial use. See our license page for the specifics..

How do I find books?

We are working on a way to easily search for books in our collection. For now, the easiest way is to use the GitHub search

I found a problem in a book, who do I tell?

  1. Find the book on github
  2. Click the "issues" tab
  3. Click "New Issue" (you may have to register for Github (and you should, github is awesome!))
  4. In the issue, copy and paste the sentence with an error as written so we can find the error in the book
  5. Describe what the text should say instead

If you follow these steps, we (or you) should have enough information to fix the book.

Can I fix issues in books?

Yes please!. This project is a collaboration between many people from all over the world. We are using the GitHub Flow workflow as our process for suggesting changes to books. These suggested changes are called Pull Requests.
